A. Executive Summary

Where We Are Today

The current AIVC GeneLink system is a prototype-grade perturbation response predictor built on a 2-layer GAT with Neumann cascade propagation. It achieves Pearson r=0.873 on Kang 2018 PBMCs (IFN-β), beating CPA (0.856) and scGEN (0.820). The orchestration layer (aivc/) introduces skill-based workflow composition, critic validation, session memory, and a structural causal model — all architecturally sound ideas.

However, the system is not production-ready and has significant gaps preventing it from being a world-class multi-omics platform.

What Changes Are Needed

To reach Insilico Medicine / Recursion Pharmaceuticals caliber: (1) separate the data platform from the model layer with a proper feature store; (2) build and validate modality-specific encoders with real multi-omics data; (3) replace file-based data handling with a scalable cell-level database; (4) implement distributed training infrastructure; (5) close the active learning loop with real wet-lab integration; (6) establish CI/CD with biological regression guards.

B. Deep Critical Review

B.1 Architecture

Critical Gap
Dual architecture with no integration. The root directory contains standalone scripts (train_v11.py, evaluate_v11.py) that operate independently from the aivc/ skill-orchestrator framework. The orchestrator's GATTrainer skill wraps train_week3.py, not the latest train_v11.py. This means the orchestrator cannot actually run the best model. The two systems have diverged.
Critical Gap
No data layer separation. Training scripts directly load .h5ad files, perform normalization inline, build graphs inline, and compute OT pairing inline. There is no distinction between raw data, processed features, and training-ready tensors. Every script re-derives everything from scratch.
Warning
Model checkpoints scattered. Model files exist in 4+ locations: root directory (model_fixed.pt, model_perturbation*.pt, model_week3*.pt), models/v1.0/, models/week4/. No model registry, no metadata linking a checkpoint to its training config/data hash.
Strength
Critic system is well-conceived. The three-critic pattern (statistical, methodological, biological) is excellent. The biological critic with JAK-STAT recovery checks and quarantine thresholds is exactly the kind of domain-aware validation that differentiates serious platforms.
Strength
Session memory with test-donor locking. The SessionMemory.lock_test_donors() pattern that prevents test set leakage is production-grade thinking — rare in academic codebases.

B.2 Model Design

Critical Gap
Multi-modal is vaporware. TemporalCrossModalFusion, ATACSeqEncoder, StructuralCausalModel, and MultiDatasetDomainAdaptation are implemented but never instantiated in any training script. Zero evidence of multi-modal training runs. The system is functionally single-modality (RNA only).
Critical Gap
No protein or phospho-protein encoder. The fusion module references protein_dim=128 and phospho_dim=64 but no ProteinEncoder or PhosphoEncoder class exists anywhere in the codebase. The ATAC encoder exists but protein/phospho are missing entirely.
Warning
GeneLink backbone is underspecified for multi-omics. The current GeneLink GAT operates on a single gene-level graph (STRING PPI). For multi-omics, you need a heterogeneous graph with peak nodes, TF nodes, gene nodes, and protein nodes — each with typed edges. The current architecture cannot accommodate this without a rewrite.
Warning
Neumann propagation doesn't scale. The Neumann series W^k with dense multiplication is O(n²) per step. At 3,010 genes this is fine. At 20,000+ genes (genome-wide) this becomes prohibitive. Need sparse implementation with edge-list-based message passing.
Strength
SCM with do-intervention is novel. The do_peak_closed() counterfactual — "what if this enhancer was silenced?" — is a compelling differentiator if connected to real ATAC data. The temporal ordering (ATAC→Phospho→RNA→Protein) is biologically sound.

B.3 Data Strategy

Critical Gap
No schema, no versioning, no lineage. Data is NumPy arrays on disk (X_ctrl_ot.npy, X_stim_ot.npy) with no metadata about which preprocessing version produced them, which gene list was used, or which cells were included. Regenerating training data requires re-running the full pipeline from scratch.
Critical Gap
Same-cell pairing is not exploited. QuRIE-seq's key differentiator is paired same-cell multi-omics. The current OT pairing operates within RNA only (matching ctrl↔stim cells). There is no mechanism to align RNA + protein + phospho + ATAC measurements from the same physical cell — which is the entire point of the technology.
Warning
ImmPort pseudo-cell generation is scientifically questionable. load_immport() generates 50 pseudo-single-cells per bulk sample by adding Gaussian noise (N(0, 0.1*|x|)). This creates artificial data that doesn't reflect real single-cell heterogeneity. It's a reasonable bootstrap for W pre-training only (correctly flagged via USE_FOR_W_ONLY), but needs explicit documentation of limitations.

B.4 Infrastructure

Critical Gap
No CI/CD, no containerization, no reproducibility guarantee. setup_gpu_instance.sh is a manual bootstrap script. No Docker/Podman, no Terraform/Pulumi, no GitHub Actions. A collaborator cannot reproduce results without following a manual setup guide.
Critical Gap
No experiment tracking. ExperimentalMemory saves JSON to disk — this is not MLflow, W&B, or even a SQLite database. Hyperparameter sweeps in train_v11.py produce .pt files with no queryable metadata store.
Critical Gap
No serving infrastructure. No API, no model registry, no inference endpoint. The system cannot serve predictions to downstream consumers (UI, notebooks, wet-lab recommendations).

C. World-Class Architecture (Redesign)

The following architecture is designed for a production-grade multi-omics AI platform capable of processing billions of cells across 4 modalities, with same-cell pairing as the core differentiator.

C.1 — Layer 1: Data Ingestion & Processing

Purpose

Accept raw multi-omics data from QuRIE-seq and external sources, validate, normalize, and register in the data catalog.

Components

Normalizationscran (RNA), CLR (protein), chromVAR (ATAC), median-centering (phospho)Modality-appropriate normalization with audit trail

Data CatalogDVC + custom metadata DBVersion every artifact (raw → processed → features) with full lineage

Design Decision: Same-Cell Pairing

QuRIE-seq produces physically paired measurements. Every cell gets a cell_barcode that links its RNA, protein, phospho, and ATAC profiles. This is NOT the same as computational integration (Seurat WNN, MOFA+). Same-cell data eliminates the "integration problem" entirely — the pairing is ground truth. The architecture must preserve this linkage end-to-end, from ingestion through to inference.

C.2 — Layer 2: Feature Store / Data Lake

Purpose

Store processed cell-level data at scale (billions of cells) with fast random access for training and serving.

Components

ComponentTechnologyResponsibility
Cell StoreApache Parquet on S3 (partitioned by experiment/sample) + DuckDB for queriesStore per-cell feature vectors (RNA: 2000-dim HVG, Protein: 200-dim, Phospho: 100-dim, ATAC: 800-dim chromVAR)
Graph StoreNeo4j or custom edge-list on ParquetGene-gene (STRING PPI), peak-gene links, TF-gene regulatory edges, protein-protein interactions
Embedding StorePinecone / pgvector / FAISS on S3Pre-computed cell embeddings for similarity search, atlas alignment, neighbor queries
Metadata StorePostgreSQLExperiment, sample, cell metadata. Donor info, perturbation conditions, cell type annotations.
Feature RegistryFeast or customTrack feature definitions (gene lists, normalization params, graph versions). Point-in-time correctness for training vs. serving.

Design Decision: Parquet over HDF5

AnnData/h5ad is convenient for small datasets but fails at scale: no column pruning, no predicate pushdown, no partition-level skipping. Parquet with Hive-style partitioning (experiment_id/sample_id/) enables reading only the cells needed for a training batch. DuckDB provides SQL over Parquet with zero-copy reads.

Fusion: Temporal Cross-Modal Attention

The existing TemporalCrossModalFusion design is sound. Key modifications:

ModificationRationale
Add causal masking to attention: future modalities cannot attend to pastEnforces biological temporal order. ATAC (t=0) is causal; protein (t=3) is effect. Protein should not influence ATAC prediction.
Add modality dropout (randomly zero a modality during training)Enables graceful degradation when a modality is missing at inference time. Critical for real-world deployment where not every cell has all 4 measurements.
Replace perturbation embedding with perturbation-aware cross-attentionFixed embedding for 10 perturbations doesn't scale. A learned perturbation representation (from gene targets, dosage, time) generalizes to unseen perturbations.

E. High-Level ERD

AIVC GeneLink — Entity Relationship Diagram (Billion-Cell Scale)

ERD Implementation Notes

Same-Cell Linkage (The Key Differentiator)

The CELL entity is the anchor for same-cell pairing. When QuRIE-seq produces measurements, each physical cell gets one cell_id. Its RNA, protein, phospho, and ATAC measurements are stored as separate rows in MEASUREMENT, all sharing the same cell_id. A simple join on cell_id reconstructs the full multi-omics profile. This is fundamentally different from computational integration where pairing is inferred — here, the pairing is physically guaranteed.

Scalability to Billions

MEASUREMENT is the largest table (~4B rows for 1B cells × 4 modalities). Partitioning strategy: hash-partition on cell_id % 1024 for parallel reads. Parquet columnar format with dictionary encoding for sparse RNA data achieves ~10:1 compression. DuckDB enables SQL queries directly over Parquet without loading into a database.

Feature Graph vs. Measurement

The FEATURE_GRAPH_EDGE table stores biological relationships between features (genes, proteins, peaks). This is separate from measurements — it represents prior knowledge (STRING PPI, PhosphoSitePlus, peak-gene links). During training, this graph is loaded once and reused. The edge types enable heterogeneous graph construction for the GAT encoder.

F. Top 5 Improvements to Reach World-Class

1 Build and Validate Real Multi-Modal Encoders

The single highest-impact change. Today, multi-modal code is vaporware — written but untrained. Implement ProteinEncoder (MLP + cross-attention to RNA), PhosphoEncoder (MLP + kinase-substrate graph), connect them through the fusion module, and train on actual paired QuRIE-seq data. Without this, the platform has no differentiator over scGPT, Geneformer, or CPA.

Success metric: Multi-modal model outperforms RNA-only model on perturbation prediction (Pearson r delta > 0.02). Cross-modal prediction loss (predict protein from RNA) converges below random baseline.

2 Implement Same-Cell Contrastive Learning

QuRIE-seq's physical pairing is unused. Implement paired contrastive loss: for each cell, its RNA embedding and protein embedding should be closer to each other than to embeddings from other cells. This creates a shared latent space grounded in biological reality — not computational integration. Add cross-modal prediction tasks (RNA→protein, protein→RNA) to learn directional relationships.

Success metric: Cell embeddings from different modalities cluster by cell identity (not modality). Cross-modal prediction R² > 0.5 on held-out cells.

3 Replace File-Based Data with Scalable Feature Store

Move from scattered NumPy arrays and h5ad files to a proper data platform: Parquet on S3 for cell measurements, PostgreSQL for metadata, DVC for versioning and lineage. Implement streaming data loaders that read directly from Parquet (no full-dataset-in-memory). This unblocks scaling from 24K cells to millions without code changes.

Success metric: Training starts within 60 seconds regardless of dataset size. Complete data lineage from raw input to model checkpoint. Any team member can reproduce results with a single command.

4 Establish ML Infrastructure (Experiment Tracking + CI/CD + Serving)

Integrate MLflow (or W&B) for experiment tracking. Add CI/CD with GitHub Actions: every PR runs unit tests + biological regression guards (Pearson r ≥ 0.873, JAK-STAT ≥ 8/15). Containerize with Docker. Deploy model serving via FastAPI + Triton with model registry for versioned deployment. This converts research code into a production platform.

Success metric: Zero manual steps from code change to deployed model. Regression in biological metrics blocks merge. Model A/B testing possible in production.

5 Close the Active Learning Loop with Wet Lab Integration

The active learning memory system exists but has no real-world connection. Implement uncertainty quantification (MC Dropout over the fused embedding), connect the experiment recommender to a LIMS/Benchling queue, and build the data-return pipeline (new QuRIE-seq data auto-triggers ingestion → feature computation → incremental fine-tuning). This creates a flywheel: model uncertainty drives experiments, experiments improve the model.

Success metric: Model uncertainty decreases monotonically over active learning iterations. Each wet-lab cycle improves JAK-STAT recovery by ≥1 gene. Recommendation-to-data time under 2 weeks.
